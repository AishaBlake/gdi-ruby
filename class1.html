What is programming?

  • Teaching the computer to do a task
  • A program is made of one or more files of code, each of which solve part of the overall task
  • Programming code is human readable but also needs a form that the computer can run directly. This form is not human readable.
  • To create the form of code the computer can use, we use the Ruby interpreter. Other languages use other interpreters or a compiler
  • Don't focus on what's "under the hood" for now. We will "drive the car" first
  • In other words, there are many layers to the onion. We start at one layer and slowly move toward layers that are beneath or above us

Command line, IRB, Text Editors

Terminal A program that has a command line interface and issues commands to the operating system.
IRB A command line program that runs inside of the terminal, takes Ruby code as input, interprets it, and prints out any results.
Text Editor A program that opens text files and allows the user to edit and save them. (Different than a word processor).

Variables

In Ruby, any lowercase word is a variable. Variables are like nicknames given to information that will be used often.

number = 5 assigns a value of 5 to the `number` variable.

There are 5 types of variables in ruby. For now, we will be using what are called ‘local variables’.

Local variables begin with a lowercase letter and may include numbers and underscores.


x = 6
number5 = 5
other_number = 2
result = x + number5 + other_number

Data types

Built-in: Numbers (Fixnum & Float), Booleans, Strings, Symbols, Regular Expressions, Arrays, Ranges & Hashes

  • Objects always have a "type"
  • The type of an object helps define what it can do.
  • The type can be found using: .class
  • An identifier for the object can be found using: .object_id

irb> number = 1
=> 1
irb> number.class
=> Fixnum
irb> apple = number
=> 5
irb> apple.class
=> Fixnum
irb> number.object_id
=> 11
irb> apple.object_id
=> 11
irb> "Hello".class
=> String
irb> 1.0022.class
=> Float
irb> true.class
=> TrueClass

Symbols

You can think of symbols as lightweight Strings.


"hello"  # transient and mutable
:hello   # permanent and immutable

It's good to know these exist, but don't think about them too much for now.

Booleans

In Ruby, true and false are reserved words that represent TrueClass and FalseClass Objects.

You can perform boolean logic with these classes: conjunction, disjunction and negation.

Regular Expression

These are one of the most powerful programming concepts ever. I'm still learning them 5 years in.

Regular Expressions are used to match patterns within Strings


irb> co_matcher = /co/
irb> puts "Match found!" if ("cool beans" =~ co_matcher)

irb> puts "The matching string was #{$~}" if "one two three four" =~ /(one).*(f...)/

Methods

  • These different objects can be used with a set of methods
  • A method is a series of statements
  • To see what methods are on an object use `.methods`
  • `.to_*` are built in methods that convert objects to a different type.
  • `.to_s` converts one object into its String representation.
  • What happens if we try to use division or subtraction with a string?

irb> "2".to_i
=> 2
irb> 2.to_s
=> "2"
irb> "2" / 5
NoMethodError: undefined method `/' for "2":String

Errors

  • There are different kinds of errors that can occur. We just saw the NoMethodError, but there are some other common errors.
  • In general, invalid operations result in a RuntimeError. You can create these by writing 'raise' in your code.
  • Another type of exception is an IOError. You might see this if you are trying to read from or write to a file that doesn't exist.
  • You will also run into the ArgumentError This comes up when you pass the wrong information into a method.

Errors - continued ...


irb> raise
RuntimeError:

irb> File.open("/etc/hosts") {|f| f << "example"}
IOError: not opened for writing

irb> def foo
irb> end
=> nil
irb> foo 1,2
ArgumentError: wrong number of arguments (2 for 0)

There are also semantic errors. These are harder to catch because the computer can't catch them for us.

Creating a folder

We need a folder to save our work in.

The ->'s below indicate the expected output of the previous command.


pwd
-> /home/username
mkdir Projects
cd Projects
mkdir gdi-ruby
cd gdi-ruby
pwd
-> /home/username/Projects/gdi-ruby

Now that the folders are made, we only have to use 'cd Projects/gdi-ruby' in the future.

User Input

To obtain user input, use `gets`

Change the class1.rb text to the following and run it again


puts "Enter a radius:"
radius = gets.chomp.to_f
area = 3.14159 * radius ** 2
puts "The area of a circle with radius " + radius + " is:"
puts area

N.B. - The user's input is a string so we use to_f make it a number
