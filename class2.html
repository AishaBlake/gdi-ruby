Boolean Expressions

We can tell the computer to compare values and return True or False. These are called Boolean expressions

  • Test for equality by using `==`. We can't use `=` because that is used for assignment
  • Test for greater than and less than using `>` and `<`

a = 5
b = 5
puts a == b
# Combine comparison and assignment
c = a == b
puts c

puts 3 < 5

Boolean Expressions continued

A boolean expression evaluates to true or false.

a == b a is equal to b
a != b a does not equal b
a < b a is less than b
a > b a is greater than b
a <= b a is less than or equal to b
a >= b a is greater than or equal to b 

a = 3
b = 4
puts a != b
puts a <= 3
puts a >= 4

Remember: Equals does not equal "equals equals"

Boolean Expressions continued, cont'd

A boolean expression evaluates to true or false. It can also have multiple parts, joined by AND (&&) or OR (||).

true and true true && true
true and false true && false
1 == 1 and 2 == 1 1 == 1 && 2 == 1
true or 1 == 1 true || 1 == 1
1 == 1 or 2 != 1 1 == 1 || 2 != 1
not (true and false) not (true && false)

Predicates and the logical operators that act on them can be wrapped in parenthesis to enforce precedence.

Conditionals

When we want different code to execute depending on certain criteria, we use a conditional

We achieve this using if statements and boolean expressions.


if x == 5
    puts 'x is equal to 5'
end

We often want a different block to execute if the statement is false. This can be accomplished using else.


if x == 5
    puts 'x is equal to 5'
else
    puts 'x is not equal to 5'
end

Conditionals continued

The following shows some examples of combining these techniques:


# 1. Both 
if x > 3 && y > 3
    puts 'Both values are greater than 3!'
end

# 2. Either 
if x != 0 || y != 0
    puts 'The point x,y is not on the x or y axis'
end

# 3. Not
if not(x > y)
    puts 'x is less than y'
end

Chained conditionals

Conditionals can also be chained.

A conditional statement can only have one 'else' statement, so chained conditionals use elsif as an additional check after the preceding `if` predicate was False.

For example:


if x > 10
   puts "x is greater than 10"
elsif x <= 10 && x!=0
   puts "x is a number between 1 and 10"
else
   puts "Wow, don't be so negative, dude"
end

Nested conditionals

Conditionals can also be nested

Nested conditionals occur inside of other conditionals and are indented over once more. When the code block is complete, they are unindented


if x > 5
  puts 'x is greater than 5'
  if x > 10
    puts '...it is also greater than 10'
  puts 'Done evaluating the x > 10 block'
puts 'Done evaluating the x > 5 block'

Loops

It is often useful to perform a task and to repeat the process until a certain point is reached.

The repeated execution of a set of statements is called iteration, or, more commonly, a loop.

One way to achieve this, is with the while loop.


x = 10

while x > 0
  puts x
  x = x - 1
end 

puts 'Done'

The while statement takes a condition, and as long as it evaluates to true, the code block beneath it is repeated.

This creates a loop. Without the `x = x - 1` statement, this would be an infinite loop :( :( :(

While loops

Consider the following example that uses iteration to derive a factorial

(A factorial of a number is equal to that number * every positive integer less than that number. E.g. The factorial of 4 is 4 * 3 * 2 * 1, which equals 24


puts 'Enter a positive integer:'

input_value = gets.chomp
#.to_i converts string to an integer
n = input_value.to_i

result = 1

while n > 1
  result = result * n
  n = n - 1
end 

puts "The factorial of " + input_value + " is:"
puts result

N.B. - This implementation does not work for negative numbers. Why?

Each loops

It is also useful to loop through a collection of elements, visiting each one to do some work, then stopping once all elements are processed.

This can be accomplished using the .each method.

First, we need a collection. We use a range of numbers to loop over.


(0..5).each do |i|
   puts "Value of i is #{i}"
end

More information about ranges can be found here.

"#{i}" is an example of string interpolation

Each loops continued

Let's examine the example carefully


(0..5).each do |i|
   puts "Value of i is #{i}"
end

The loop has three parts:

  • The collection that will be looped through, in parenthesis because it's a range- "(0..5)"
  • The name to give each element when the loop begins again - "i"
  • The code to execute with the element - The puts statement

Collections

There are two main types:

  • Arrays
  • Hashes

new_array = [1, 3, 5]

new_hash = {1 => "snoopy", 3 => "woodstock", 6 => "pigpen"}

Arrays


new_array = [1, 3, 5]
arr = ["wow", "woooo", "zowie"]

Arrays are ordered

Arrays are integer-indexed


new_array[0] #returns 1, the zeroth item in this array

arr[2] #returns "zowie", the third item in this array

arr[-1] #returns "zowie", the last (1st from the end) item in this array

new_array.last #returns 5, the last item in this array

arr.first #returns "wow", the first item in this array

Learn more about arrays here.

Hashes


grades_hash = { "Jane Doe" => 10, "Jim Doe" => 6, "Jan Doe" => 8}

new_hash = { 1 => "a", "d" => 12, "f" => 35 }

Hashes are unordered.

Hashes are like dictionaries, with unique keys and their values.


grades_hash["Jane Doe"] #returns 10, the value of this key

new_hash["d"] #returns 12, the value of this key

Learn more about arrays here.

Adding Elements


grades_hash = { "Jane Doe" => 10, "Jim Doe" => 6, "Jan Doe" => 8}

arr = ["wow", "woooo", "zowie"]
                    

grades_hash["Juno Doe"] = 7
# grades_hash changed to { "Jane Doe" => 10, "Jim Doe" => 6, "Jan Doe" => 8, "Juno Doe" => 7}
# the new key/value pair has been added to the hash

arr.push("yikes")
# array changed to ["wow", "woooo", "zowie", "yikes"]
# the new element has been pushed into the last spot in the array

arr[2] = "gadzooks"
# array changed to ["wow", "woooo", "gadzooks", "yikes"]
# the third element has been reset to "gadzooks"
